# さんすうクイズ

小学1〜6年むけの **算数（計算）クイズ集**。漢字クイズ（`~/kanji-quiz`）の UI/UX・育成システムを参考に新規作成。
ブラウザで開くだけ（`file://` でも GitHub Pages でも動く）。スマホ・タブレット・PC 対応。

- **遊ぶ（公開URL）**: https://mikushiba.github.io/sansu-quiz/
- **リポジトリ**: https://github.com/mikushiba/sansu-quiz （漢字クイズとは独立リポジトリ）
- **本体（編集場所）**: `~/sansu-quiz/`
- 進捗データは漢字クイズ（`kanjiQuiz.v1`）とは別オリジン・別ストアなので **混ざらない**。

---

## あそびかた

トップで **学年（1〜6年）→ クイズ** をえらぶ。各クイズは：

1. **レベル**をえらぶ（くり上がり/くり下がりの有無、けた数）
2. **もんだいの数**をえらぶ（10/20/50/ぜんぶ）
3. **テンキー**で答えを入れて「こたえる」（PCは数字キー＋Enterでも可）
4. まちがえると **ヒント** が出る／2年以降は **手書きメモ** で筆算できる
5. 連続正解で **🔥コンボ**、最後に **点数・⭐・育成のごほうび**

## トップ（学年カテゴリ・計算のみ）

トップは **1年生〜6年生** ごとに「その学年で習う計算」を並べる。実装済みは遊べ、未実装は **「🔜 じゅんびちゅう」** 表示（押せない）。リンクは `?lv=<レベルID>` でクイズの初期レベルを指定。

| 学年 | 実装済み（遊べる） | じゅんびちゅう 🔜 |
|---|---|---|
| 1年 | たしざん／ひきざん | — |
| 2年 | たし算ひっ算(2けた)／ひき算ひっ算(2けた) | かけ算（九九） |
| 3年 | 大きなたし算/ひき算(3〜4けた) | かけ算筆算／わり算／小数・分数 |
| 4年 | — | わり算筆算／小数(±,×÷整数)／分数(同分母) |
| 5年 | — | 小数×÷小数／異分母分数／約数倍数 |
| 6年 | — | 分数×÷分数／文字式 |

> 学年見出しは **トップだけ**。各クイズ内のレベル選択には学年見出しを出さない（二重表示を避けるため）。

## たし算クイズ（`tashizan/`）

- **レベル（8段。くり上がりの有無で刻む）** ＝ レベルID:
  - `add10` 10までの たしざん（答え≤10）
  - `carry1` くり上がりの たしざん（1けた＋1けた・答え11〜18）※標準
  - `mix20` 20まで ミックス
  - `d2p1n` / `d2p1c` 2けた＋1けた（くり上がり なし／あり）
  - `d2p2n` / `d2p2c` 2けた＋2けた（くり上がり なし／あり）
  - `d3p3` 3けた＋3けた
- **くり上がりヒント**（`makeTenHint`）：まちがえると「10のかたまり（さくらんぼ）」で考え方を提示
  例）`8に2をたして10、のこり4で14`
- 多けたレベルは **くり上がり あり/なし を生成で厳密に出し分け**（混在させない）。

## ひき算クイズ（`hikizan/`）

- **レベル（8段。くり下がりの有無で刻む）** ＝ レベルID:
  - `s10n` ひきざん（くり下がり なし・1けたどうし）
  - `sb` くり下がりの ひきざん（十何−1けた）※標準
  - `s20mix` 20まで ミックス
  - `d2m1n` / `d2m1b` 2けた−1けた（くり下がり なし／あり）
  - `d2m2n` / `d2m2b` 2けた−2けた（くり下がり なし／あり）
  - `d3m3` 3けた−3けた
- **くり下がりヒント**（`makeBorrowHint`・減加法）：十何−1けたで
  例）`15を 10と 5に わけて、10−8＝2、2＋5＝7`
- 生成は **つねに a≧b**（答えがマイナスにならない）。くり下がり あり/なし を厳密に出し分け。

## 共通の作り（両アプリ）

- **テンキー入力**：けた数はレベルに連動（`maxDigits`）。式が長い3けたは自動で文字を縮小（`.eq-sm`）。
- **手書きメモ（けいさんメモ）**：2年以降（`memo:true`）でノート罫線つきキャンバスを表示。指/ペン/マウスで筆算でき、問題ごとに自動クリア。高さ340px。
- **デイリーチャレンジ**：日付シードで毎日同じ10問。
- **まちがい復習**：まちがえた式だけ もう一回。
- **音**：Web Audio（ファイル不要・オフライン可）。ON/OFF切替。

## 育成システム（`shared/sansu-core.js` ＝ SansuCore）

漢字クイズと同じ思想で、**全クイズ（たし算・ひき算…）で進捗を合算**。

- **📖 けいさんカードずかん**：1〜9のたし算カード 全45まい（交換は同一）。正解で図鑑入り→3回で★マスター。
  - ※ 現状カードは **たし算のみ** たまる。ひき算は称号・メダル・コンボ・累計には加算されるが、図鑑カードは増えない（将来ひき算カードを追加可）。
- **🐣 称号17段**：けいさん みならい → … → けいさんの神さま（累計正解数で成長）。
- **🏅 メダル 約95種**：あそぶ回数／全問正解／累計正解／コンボ／なおし／図鑑／マスター／デイリー連続日数 など。
- **🔥 コンボ・紙ふぶき演出**、**💾 きろくの ほぞん／よみこみ**（JSON・全クイズ分まとめて）。
- 進捗 localStorage キー: **`sansuQuiz.v1`**（オリジン単位で全クイズ共有）。
  - アプリ専用フィールドは名前空間分離：`tashizanWrong`/`tashizanDaily`、`hikizanWrong`/`hikizanDaily`。他アプリのフィールドは壊さない（full-load→一部更新→full-save）。

## 自動アップデート（キャッシュ対策）

「中身（インラインJS）だけの更新」は `?v=` では届かない（HTML自体がキャッシュされるため）。そこで：

- 各HTML先頭で `shared/build.txt`（版番号）を **`cache:'no-store'`** で取得。
- 自分の `BUILD` 定数より新しければ **1回だけ** `?_v=<版>` を付けて取り直す（`sessionStorage` でループ防止）。
- `file://` では fetch が失敗するだけで無害（何もしない）。

## 公開（デプロイ）手順

GitHub Pages は **main ブランチ直下** を配信（漢字クイズと同方式・`build_type=legacy`）。

```bash
cd ~/sansu-quiz
# 1) 変更する
# 2) 版番号を新しい値に統一（自動アップデートを効かせる）
#    - shared/build.txt
#    - 各HTMLの BUILD='...'（index/tashizan/hikizan）
#    - 各HTMLの ?v=...（任意。CSS/JSのキャッシュ更新用）
git add -A && git commit -m "…"
git push origin main
# 3) Pages ビルド完了を待ち、公開URLで確認
gh api repos/mikushiba/sansu-quiz/pages/builds/latest --jq .status   # → built
```

> 版番号は `YYYYMMDDhhmm` 形式（例: `202606270600`）。**build.txt と各HTMLの BUILD を必ず同じ値に揃える**こと。
